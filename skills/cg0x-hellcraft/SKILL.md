# Hellcraft — 地狱笑话锻造术

生成结构严谨的地狱笑话。不是随便说一句冒犯的话——是用精确的语言机械制造认知跳轨，让听者在“get到”的瞬间同时体验到笑和不适。

## 命令

```
/cg0x-hellcraft
```

无参数。执行后直接生成 **1 则**地狱笑话，不做解释。像零食一样——随手一条，乐完继续干活。
如果用户追问“为什么好笑”或要求解析，再附加结构拆解。

**语言**：跟随用户的输入语言，自动切换输出语言、落点交付风格和素材知晓度偏好。用户用中文问就输出中文笑话，用英文问就输出英文笑话。不需要用户指定，不需要额外参数。

## 为什么地狱笑话有效

地狱笑话的力量不来自“冒犯”本身，而来自三个同时运作的机制：

### 机制 1：语义跳轨（结构力学）

每则笑话都是一台语言机器。设置项（setup）在听者脑中建立一条无害的语义通道，落点（punchline）迫使大脑突然跳到另一条语义轨道。跳轨距离越大，冲击越强。

好的设置项使用某个正常领域的合法词汇（算术、统计学、政治、工业、物流、军事），落点揭示这些词汇的第二层含义指向大规模死亡。

### 机制 2：施害者逻辑复现

最“地狱”的笑话不是外部嘲讽暴行，而是精确复现了施害者的真实思维方式。落点能成立，是因为历史上的施害者真的是这么想的。

听者在“get到”笑话的瞬间，被迫在自己脑内运行了一次施害者的推理过程。笑和理解暴行的内部逻辑是同一个认知动作。

### 机制 3：去人化转换

落点要求听者临时采用一种将人视为非人的视角。“get到”笑话 = 在认知上完成了一次“人→物”的转换。

去人化梯度（从弱到强）：数字 → 统计量 → 抽象标签 → 工业原料 → 无生命货物 → 食物。越靠后冲击越大。

## 生成步骤

### Step 1：选定历史素材

收到用户请求后，选择真实历史暴行事件作为素材。虚构暴行的笑话传播力弱——缺少“这是真的”这层冲击。

素材选择标准：
- 必须是有据可查的真实历史事件
- 事件中存在可辨认的“施害者逻辑”（如何将暴行合理化/技术化/官僚化）
- 事件涉及某种“去人化”操作（将人归类为非人类别）
- 事件有足够的公众知晓度（听者能“get到”）

素材库参考（不限于此）：
- 大屠杀与种族灭绝（Holocaust、卢旺达、红色高棉、亚美尼亚）
- 人造饥荒与政策性灾难（乌克兰大饥荒、大跃进、孟加拉饥荒）
- 恐怖袭击与战争罪行（9/11、广岛、东京大轰炸、德累斯顿轰炸）
- 奴隶制与殖民暴行（大西洋奴隶贸易、刚果自由邦、美洲原住民灭绝）
- 古代征服（蒙古西征、十字军、亚述帝国）

### Step 2：提取施害者逻辑

找到施害者（或体制）是如何在内部语言中描述这件事的——他们用了什么委婉语、什么技术术语、什么管理框架来处理大规模死亡。

关键问题：
- 施害者把人叫什么？（“货物”、“件”、“要素”、“问题”）
- 施害者把杀戮叫什么？（“最终解决方案”、“清洗”、“再教育”、“转移”）
- 施害者用什么系统来管理暴行？（物流系统、统计报表、工业流程、军事后勤）
- 这些术语在哪个正常领域也能成立？

这一步决定了笑话的“地狱深度”——越贴近真实施害者语言，越地狱。

### Step 3：构建双轨结构

笑话 = 设置项 + 落点，两条语义轨道共用同一组词汇。

**设置项**：用正常领域的合法提问方式包装。听者应该在读完设置项时处于一条无害的语义轨道上。

设置项的常见句式：
- “为什么 [X]？”（疑问句式）
- “[X] 如何实现 [Y]？”（方法论句式）
- “[X] 为何从不/总是 [Z]？”（规律句式）

**落点**：用最短的文字完成语义跳轨。落点必须同时满足：
1. 在无害轨道上说得通（表面合理）
2. 在暴行轨道上精确指向真实历史（深层指涉）
3. 两条轨道之间的距离尽可能大

落点越短越好。最好不超过一个短句。信息密度要高到听者需要一拍才能反应过来。

#### 中英文落点交付差异

中文和英文的地狱笑话在落点交付上有本质不同，根据输出语言自动切换：

**中文 — 断裂式**：落点不用"因为"，直接砸答案。中文每字语义密度高，裸落点像铁块——听者被迫自己拼因果，这个拼的过程就是笑点。
- ✘ "纳粹列车为何从不延误？因为货物保质期短"
- ✔ "纳粹列车为何从不延误？货物保质期短。"

**英文 — 节拍式**："Because" 可以保留，但功能不是逻辑连接，而是节拍标记——它制造一个半拍的悬念，告诉听者"下一个词就是落点"。
- ✔ "Why were Nazi trains never late? Because the cargo had a short shelf life."
- ✔ "Why were Nazi trains never late? The cargo had a short shelf life." （也可以裸落点，取决于节奏）

#### 中英文双关机制差异

根据输出语言选择不同的双轨构建策略：

| 策略 | 中文优先用 | 英文优先用 |
|------|-----------|-----------|
| **典故压缩** | ✔ 四字格/历史用语自带双轨（"两脚羊"、"三年自然灾害"） | 弱——英文缺少等密度的典故系统 |
| **官僚委婉语** | ✔ 政治委婉语（"再教育"、"上山下乡"） | ✔✔ 英文公文体是双轨机器（"Final Solution"、"collateral damage"、"enhanced interrogation"） |
| **谐音双关** | 可用但慎用——谐音梗容易拉低地狱感 | 中等（"deadline"、"executed"） |
| **词根双关** | 不适用 | ✔ 拉丁/日耳曼双源词根天然双义 |

#### 素材知晓度差异

不同语言社群的"常识暴行"不同，选素材时要匹配受众：

| 中文母语者更熟 | 英文母语者更熟 | 双方都熟 |
|---|---|---|
| 大跃进、文革、南京大屠杀、731部队、蒙古西征 | Holocaust、大西洋奴隶贸易、刚果自由邦、亚美尼亚种族灭绝 | 9/11、广岛、红色高棉、卢旺达、乌克兰大饥荒 |

如果用户用中文提问，优先选中文母语者更熟或双方都熟的素材。英文同理。混合受众场景（如用户明确要求双语）优先选双方都熟的素材。

### Step 4：校验三层机制

生成后逐条检查：

| 检查项 | 问题 | 不通过则 |
|--------|------|---------|
| **跳轨距离** | 设置项和落点之间的语义距离够大吗？ | 加大领域跨度 |
| **施害者真实性** | 落点的逻辑是否贴近施害者的真实思维/语言？ | 查证历史细节 |
| **去人化层级** | 人在笑话中被转换成了什么？层级够高吗？ | 考虑升级到更具身的类别 |
| **可“get到”性** | 不做额外解释，听者能在3秒内理解吗？ | 简化或换更知名的事件 |
| **压缩度** | 整个笑话能在一两句话内完成吗？ | 砍字 |

### Step 5：输出

**默认输出**：直接给出笑话，不做解释。地狱笑话的力量在于听者自己“get到”。

如果用户要求解析，则对每则笑话附加：
- **历史锚点**：引用的真实事件
- **施害者语言**：哪部分复现了施害者的真实逻辑
- **去人化类别**：人在笑话中被转换成了什么
- **跳轨距离**：从哪个正常领域跳到哪个暴行

**数量**：默认 1 则。每次调用随机选取不同的历史素材和去人化层级，保证连续调用不重复。

## 常见失败模式

1. **纯冒犯无结构**："XX 人都该死"——这不是笑话，只是辱骂。没有双轨，没有跳轨。
2. **虚构历史**：编造不存在的暴行——失去"这是真的"这层冲击。
3. **落点过长**：需要三句话才能解释落点——压缩度不够，失去即时性。
4. **施害者逻辑缺失**：笑话从外部视角嘲讽，而非从施害者的内部逻辑出发——少了最地狱的那层。
5. **去人化过弱**：人在笑话中仍然被当作人——冲击打折。
6. **事件太冷门**：听者需要先去百科补课才能"get到"——即时性丧失。
